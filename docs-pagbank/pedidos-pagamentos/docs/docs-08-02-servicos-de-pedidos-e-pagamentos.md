# Serviços de pedidos e pagamentos

Esta página reúne, em linguagem de negócio, os principais recursos da API de pedidos e pagamentos.

## Criação de pedidos e pagamentos

A API pode ser usada para criar e operar pedidos com diferentes métodos de pagamento.

## Pagamento com Cartões

O PagBank suporta pagamento com cartão de crédito e débito.

### Utilizando tokens para pagar com cartão

Você pode tokenizar cartões para reutilização segura em novas cobranças.

### Autenticação 3DS

O fluxo 3DS adiciona uma camada extra de autenticação para aumentar a segurança da cobrança.

## Pagamentos com Boleto Bancário

O boleto é uma alternativa de pagamento para pedidos que não dependem de autorização instantânea.

## Pagamentos com PIX

PIX também pode ser usado como forma de pagamento dentro do fluxo de pedidos.

## Utilizando o recurso de Pagar com PagBank

Em cenários com checkout ou experiência guiada, o Pagar com PagBank pode simplificar a etapa de pagamento.

## Divisão do pagamento

Use split para distribuir valores entre recebedores.

## Cancelamento de pagamentos

A API também contempla cancelamento de cobranças autorizadas e reembolsos.

### Cancelamento de cobranças autorizadas

Use este fluxo quando a cobrança já foi autorizada, mas ainda pode ser cancelada.

### Reembolsos parciais

Alguns cenários permitem reembolso parcial, dependendo do estado da transação.

### Repasse taxas ao comprador

Em determinados modelos, é possível repassar taxas ao comprador.

## Suporte

Se tiver dúvida de implementação, use esta página como guia conceitual e consulte a referência técnica dos endpoints.
